MagiColorLike

Creative Tool / Editor
UnityC#
Solo

What is it?

An interactive character customisation tool built in Unity. A white robot character is displayed on screen and users can upload any image to apply it as the robot's texture. The UV map of the robot can also be exported as a printable colouring sheet — so players can draw their own skin by hand, photograph it, and apply it in the app.

Key technical features

  • Runtime image upload pipeline: reads an image from disk and maps it to a 3D mesh's UV coordinates at runtime
  • UV map export as a printable PDF/image
  • Texture swapping on a rigged character without breaking materials or normals
  • Clean UI for upload, preview, and reset

What makes this unusual

This project was more focused as a tool or a game feature. The UV-based workflow is similar to what professional games use for custom character skins. Handling runtime texture replacement correctly — without memory leaks or material conflicts — was the main technical challenge.

Tower Defence 3D

Strategy / Tower Defence
UnityC#
👥 Collab

What is it?

A 3D tower defence game with a fully plug-and-play wave system. Zombies advance toward a protected zone across multiple rounds — players place and upgrade towers to stop them. The game spans multiple rounds in different locations, each containing multiple waves.

Core systems

  • Modular wave system: add new enemy waves by configuring a data object, no code changes needed
  • Four tower types — melee, ranged, and specialist variants — each with three upgrade tiers
  • Tower placement grid with valid/invalid zone detection
  • Currency system: earn gold per kill, spend on placement and upgrades
  • Round and wave progression with clear UI feedback

The plug-and-play wave system

The wave system was designed to be data-driven rather than hardcoded. Each wave is defined in a ScriptableObject — enemy types, counts, spawn delay — meaning new waves can be created without touching any scripts. This pattern helped significantly reduces the risk of bugs when adding content.

Target Trainer 3D

FPS Mini-game
UnityC#
Solo

What is it?

A fast-paced aim-training mini-game. Players stand on a weapon range and work through two zones: a target range where pop-up targets fall and reset when hit, and a room clear where bottles and reactive props can be destroyed. Speed and accuracy drive the score — missing deducts points. A leaderboard tracks high scores across sessions.

Everything was self-built

  • First-person player controller with mouse-look and weapon handling
  • Shooting system with raycasts, hit detection, and miss penalty logic
  • Reactive targets: pop-up targets that fall on hit and reset after a delay
  • Destructible props: bottles and objects with physics reactions on impact
  • Score system with speed multiplier — faster hits earn more
  • Persistent leaderboard saving top scores between sessions
  • Full animation set for weapon draw, fire, and reload

Design decision: miss penalty

Adding a deduction for missed shots was intentional. It makes the game about precision, not just speed, and raises the skill ceiling significantly. Small design decisions like this have a big effect on how a mechanic actually feels to play.
